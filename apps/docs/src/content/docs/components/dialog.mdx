The `Dialog` component (also known as a modal) provides a way to display information and collect input without leaving the current context. It is built on top of [Bits UI](https://bits-ui.com/docs/components/dialog).

## Usage

Import the `Dialog` components and use them to build your modal.

<Tabs>
  <TabItem label="Svelte">
```svelte
<script>
  import { Dialog, Button } from "@adn-ui/svelte";
</script>

<Dialog.Root>
  <Dialog.Trigger>
    <Button variant="outline">Open Dialog</Button>
  </Dialog.Trigger>
  <Dialog.Overlay />
  <Dialog.Content>
    <Dialog.Header>
      <Dialog.Title>Edit profile</Dialog.Title>
      <Dialog.Description>
        Make changes to your profile here. Click save when you're done.
      </Dialog.Description>
    </Dialog.Header>
    <!-- Content goes here -->
    <Dialog.Footer>
      <Button type="submit">Save changes</Button>
    </Dialog.Footer>
  </Dialog.Content>
</Dialog.Root>
```

## API Reference

### Dialog.Root

The root component that manages the dialog state.

| Prop           | Type                      | Default     | Description                                 |
| :------------- | :------------------------ | :---------- | :------------------------------------------ |
| `open`         | `boolean`                 | `false`     | The controlled open state of the dialog.    |
| `onOpenChange` | `(open: boolean) => void` | `undefined` | Callback fired when the open state changes. |

### Dialog.Trigger

The element that opens the dialog. Usually a `Button`.

### Dialog.Overlay

The semi-transparent backdrop that covers the page content when the dialog is open.

### Dialog.Content

The container for the dialog content.

### Dialog.Header

A wrapper for the title and description, providing consistent spacing.

### Dialog.Title

The title of the dialog, which is announced by screen readers.

### Dialog.Description

An optional description for the dialog, also announced by screen readers.

### Dialog.Footer

A container for actions, typically buttons, at the bottom of the dialog.

### Dialog.Close

A wrapper for elements that should close the dialog when clicked.

## Accessibility

The `Dialog` component adheres to the [WAI-ARIA Dialog (Modal) Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/):

- **Focus Management:** Automatically traps focus within the dialog and restores it to the trigger upon closing.
- **Keyboard Support:** Supports `Esc` to close.
- **ARIA Attributes:** Manages `role="dialog"`, `aria-modal="true"`, `aria-labelledby`, and `aria-describedby`.
